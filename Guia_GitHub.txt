# 💈 Guía de Git y GitHub - Delivery Barber 💈

¡Bienvenido al equipo! Esta guía te ayudará a sincronizar tu trabajo con el resto del equipo usando Git y GitHub. No te preocupes si eres nuevo en esto, sigue estos pasos y estarás listo en poco tiempo.

---

## 🛠️ 1. Configuración Inicial (Solo se hace una vez)

Antes de empezar, Git necesita saber quién eres para marcar tus cambios. Abre tu terminal y escribe:

```bash
# Configura tu nombre (entre comillas)
git config --global user.name "Tu Nombre Real"

# Configura tu correo electrónico de GitHub
git config --global user.email "tu_correo@ejemplo.com"
```

Para confirmar que todo está bien:
`git config --global --list`

---

## 📥 2. Cómo obtener el proyecto

Para bajar una copia del proyecto a tu computadora por primera vez:

1. Ubícate en la carpeta donde quieras guardar el proyecto.
2. Ejecuta:
   ```bash
   git clone https://github.com/D4NX3L8/DeliveryBarber.git
   ```
3. Entra a la carpeta del proyecto:
   ```bash
   cd DeliveryBarber
   ```

---

## 🔄 3. Tu Flujo de Trabajo Diario (Paso a Paso)

Sigue este orden cada vez que vayas a trabajar para evitar conflictos con tus compañeros:

### Paso A: Recibir lo nuevo
Antes de tocar cualquier código, descarga lo que otros han subido:
```bash
git pull origin master
```

### Paso B: Hacer tu magia
Modifica los archivos, crea nuevas funciones o arregla estilos en tu editor favorito.

### Paso C: Revisar qué cambió
Mira qué archivos has modificado:
```bash
git status
```

### Paso D: Preparar los cambios
Dile a Git qué archivos quieres subir (el punto `.` significa "todos"):
```bash
git add .
```

### Paso E: Ponerles nombre (Commit)
Escribe un mensaje corto y claro de lo que hiciste:
```bash
git commit -m "Explicación breve de tu cambio"
# Ejemplo: git commit -m "Agregado el botón de reserva en el inicio"
```

### Paso F: Subir a la nube
Envía tus cambios al repositorio compartido:
```bash
git push origin master
```

---

## 🔍 Comandos de Verificación

*   **¿En qué estoy trabajando?**: `git status` (Úsalo mucho, te dice si todo está al día).
*   **¿Qué se ha hecho antes?**: `git log --oneline` (Muestra una lista de los últimos cambios).
*   **¿Dónde estoy conectado?**: `git remote -v`.

---

## 📜 Reglas de Oro del Equipo

1. **Primero descarga, luego trabaja:** Siempre inicia con un `git pull`.
2. **Mensajes claros:** No uses mensajes como "cambios" o "update". Sé específico.
3. **Pequeños pasos:** Es mejor subir cambios pequeños seguido que un cambio gigante una vez a la semana.
4. **No trabajes a ciegas:** Si `git status` te muestra cosas rojas que no reconoces, pregunta antes de seguir.
5. **Comunicación:** Si vas a cambiar algo muy importante, avisa en el grupo.
